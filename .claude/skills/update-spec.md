# 仕様書更新スキル

アプリケーションコードに機能追加・変更・削除を行った際に、仕様書を最新の状態に保つ。

## 手順

1. `docs/spec.md` を読み込む
2. 今回の変更内容（git diffまたはユーザーの説明）を確認する
3. 仕様書の該当セクションを特定し、以下を更新する:
   - 新機能の場合: 「機能詳細」セクションに新しいFn項目を追加
   - 変更の場合: 該当する既存セクションの記述を修正
   - UI変更の場合: 「UI構成」セクションも更新
   - データ構造変更の場合: 「データ構造」セクションも更新
   - 拡張候補が実装された場合: 「今後の拡張候補」から該当行を削除し、機能詳細に移動
4. 変更内容をユーザーに要約して報告する
