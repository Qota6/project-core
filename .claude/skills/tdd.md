# TDD（テスト駆動開発）スキル

仕様書 `docs/spec.md` に基づいてテスト駆動開発を行う。

## ワークフロー

### 1. RED — テストを先に書く
- `docs/spec.md` の該当セクションを読む
- 仕様の振る舞いをテストケースとして記述する
- テストを実行し、**失敗すること（RED）** を確認する

### 2. GREEN — 最小限の実装を書く
- テストを通すための最小限のコードを実装する
- テストを実行し、**全て通ること（GREEN）** を確認する

### 3. REFACTOR — リファクタリング
- テストが通る状態を維持しつつコードを整理する
- テストを再実行し、**通ること** を確認する

## ルール

- テストは仕様書の機能番号（F1〜）やセクション名で整理する
- テストは実装の内部構造に依存しない（ブラックボックス）
- 各ステップでテストを実行し、RED→GREEN→REFACTORのサイクルを明示する
- コアロジック（packages/core）のテストは `vitest` の `node` 環境で実行
- Webコンポーネントのテストは `jsdom` 環境 + `@testing-library/react` で実行

## テスト実行コマンド

```bash
pnpm test          # 全テスト
pnpm test:core     # コアロジックのみ
pnpm test:web      # Webコンポーネントのみ
```
