---
name: テストは仕様書ベース
description: テストは実装の内部構造ではなく仕様書（docs/spec.md）の振る舞い記述に基づいて書く
type: feedback
---

テストは仕様書ベースで書く。実装の内部構造（関数名、内部変数、実装詳細）に依存しないブラックボックステストを基本とする。

**Why:** 実装ベースのテストはリファクタリング時に壊れやすく、仕様が変わっていないのにテスト修正が必要になる。仕様書ベースなら「何をするか」を検証するため、実装方法が変わってもテストは安定する。

**How to apply:** テストケースを書く際は、まず `docs/spec.md` の該当機能を読み、そこに記載されている振る舞いをテストする。内部実装を知っていても、テストからは公開API/関数の入出力のみを検証する。
